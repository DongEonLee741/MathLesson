## 수정 1: 에라토스테네스의 체 UX 개선

### 파일: session4_unified.html

### 1-A. 조기 하이라이트 버그 수정

**현재 문제**: `sieveStep(p)` 끝에서 `isPrime(i)` 판정으로 아직 체를 안 건 소수까지 초록색으로 변환.

**수정**: 해당 루프 완전 삭제. 소수 하이라이트는 **4개 버튼(2,3,5,7) 모두 클릭 완료 후**에만 실행.

```
삭제할 코드:
  // Mark remaining non-eliminated as prime candidates
  for(let i=2;i<=100;i++){
    if(sieveState[i]==='none'&&isPrime(i)){sieveState[i]='prime';const c=document.getElementById('sieve-'+i);if(c)c.className='sieve-cell prime';}
  }

대체 로직:
  - sieveCompleted 카운터 관리 (2,3,5,7 각각 클릭 시 +1)
  - 카운터 === 4일 때:
    1. 모든 eliminated 셀 → 회색(.eliminated-final) 통합
    2. 남은 none 상태 셀(2~100 중) → .prime 클래스 하이라이트
    3. "🎉 25개 소수를 모두 찾았습니다!" 메시지
```

### 1-B. 단계별 색상 구분

각 소수 배수 제거 시 고유 색상 적용:

| 버튼 | CSS 클래스 | 배경색 | 테두리색 |
|------|----------|--------|---------|
| 2의 배수 | `.elim-2` | #fecaca (빨강 연) | #f87171 |
| 3의 배수 | `.elim-3` | #fed7aa (주황 연) | #fb923c |
| 5의 배수 | `.elim-5` | #fef08a (노랑 연) | #facc15 |
| 7의 배수 | `.elim-7` | #e9d5ff (보라 연) | #a78bfa |
| 전부 완료 후 | `.eliminated-final` | #f3f4f6 (회색) | #d1d5db |

중복 제거 시 (이미 다른 색인 셀): 기존 색 유지, 건너뜀.

### 1-C. 순차 애니메이션

각 `sieveStep(p)` 호출 시:
- 제거 대상 셀을 배열로 모은 후, **60ms 간격**으로 하나씩 색 전환
- 전환 시 짧은 scale 애니메이션 (transform: scale(0.85) → scale(1), 200ms)
- 애니메이션 완료 후 "이번에 N개 새로 지움" 카운터 표시

### 1-D. "이번에 N개 지움" 카운터

`sieveStep` 실행마다 `#sieveStepCount` 영역에 표시:
```

"3의 배수: 16개 새로 제거"  
"5의 배수: 6개 새로 제거"
"7의 배수: 3개 새로 제거"
```

→ 학습지 E-1 빈칸과 직접 대응.

### 1-E. 버튼 순서 강제

현재: 4개 버튼이 동시에 활성화 → 학생이 순서 무시 가능.

## 수정 3: 분해 문제 값 변경

### 파일: session4_unified.html, session4_worksheet.html

### 변경 내용

**곱하기 문제 (유지)**:
```
MULT_PROBS = [{a:7, b:13, ans:91}, {a:23, b:41, ans:943}, {a:61, b:89, ans:5429}]
```

**분해 문제 (변경)**:
```
기존: [{n:91, f:[7,13]}, {n:943, f:[23,41]}, {n:5429, f:[61,89]}]
변경: [{n:77, f:[7,11]}, {n:899, f:[29,31]}, {n:1517, f:[37,41]}]
```

검증:
- 77 = 7 × 11 ✅
- 899 = 29 × 31 ✅ (√899 ≈ 30, 인접 소수 쌍)
- 1517 = 37 × 41 ✅ (√1517 ≈ 38.9)

### Worksheet 반영

E-5 분해 도전 표:
```
91 = ? × ?   →   77 = ? × ?
943 = ? × ?  →   899 = ? × ?
5429 = ? × ? →   1517 = ? × ?
```

### 비교 그래프

곱하기와 분해 문제가 더 이상 같은 수가 아니므로, 그래프 구조 변경:

**방식**: 1:1 쌍 비교 → **평균 시간 비교 + 크기별 추세**

```
왼쪽: 곱하기 3문제 막대 (초록) + 평균선
오른쪽: 분해 3문제 막대 (빨강) + 평균선
```

- 곱하기 라벨: "7×13", "23×41", "61×89"
- 분해 라벨: "77", "899", "1517"
- 하단에 평균 시간 표시: "곱하기 평균: X.Xs / 분해 평균: Y.Ys"
- info-box: "숫자 크기가 비슷해도 방향이 다르면 시간 차이가 극적입니다!"
